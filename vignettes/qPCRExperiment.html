The Experiment Object

The Experiment object attempts to collect together all the infomation from multiple EDS files. From the top-level down, the experiment contains a list of Plates which are each individual EDS file that goes into making the Experiment object. Within each Plate there are a list of Wells that contains the actual information from the experiment. The data is organised in such a way that you should only ever need to interact with the Experiment and can manipulate the data as needed without ever changing the values inside the Wells

Examplar Quality Control

The Experiment object contains a couple of varieties cycle threshold values, Namely Ct and Crt Raw. These can be accessed via the get method:

# The `deep=TRUE` argument lets the experiment object see data into the wells.
ct_vals <- example_experiment$get('Avg Ct', deep=TRUE)
plot(density(ct_vals))

crt_vals <- example_experiment$get('Crt Raw', deep=TRUE)
plot(density(crt_vals))

crt_amp <- example_experiment$get('Crt Amp', deep=TRUE)
plot(density(crt_amp))

raw_amp <- example_experiment$get('Raw Amp Score', deep=TRUE)
plot(density(raw_amp))

cq_conf <- example_experiment$get('Cq Conf', deep=TRUE)
plot(density(cq_conf))

For our analysis in xyz paper, we used a min Cq Conf of 0.8, A min Crt Amp of 0 and also limited our analyses to 0 and 40. Seeing as we extracted the raw data we can simply apply the filters to the objects in memory.

crt_vals_filtered <- crt_vals
is.na(crt_vals_filtered) <- cq_conf < 0.8 | crt_amp < 0 | crt_vals <= 0 | crt_vals >= 40
